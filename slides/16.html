Prompt Engineering

Bad vs. Good Prompts

Precision beats brevity. Add context, constraints, and criteria.

Ineffective

"Refactor this file"

Too broad, no goal
Effective

INLINE CHAT Refactor selected function into a pure function. Keep signature identical, remove I/O side effects, ensure O(n) time complexity. Provide diff + brief rationale.

Ineffective

"Write tests"

No scope or framework
Effective

@WORKSPACE Generate unit tests for parseConfig() covering empty, malformed, and missing-file cases. Use Jest, mock file system access, and avoid network I/O.

Ineffective

"Fix this error"

Lacks context
Effective

/FIX TypeError in userMapper.ts: handle null user.role scenarios by returning default 'viewer'. Include diff and add one regression test case.
