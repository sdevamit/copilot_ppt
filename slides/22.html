Custom Agents

// Declarative .agent.md or Extension API

Declarative: .github/agents/*.agent.md

Define name, tools list, model, and system prompt in YAML frontmatter + Markdown body. No extension code needed — VS Code discovers agent files automatically.

Programmatic: Extension API

Use vscode.chat.createChatParticipant(id, handler) in an extension for full control: custom LLM calls, streaming, tool invocations, and VS Code API access.

Handoffs & Subagents

Add handoffs: in frontmatter to chain agents (Plan → Implement → Review). Use agents: ['*'] to allow sub-agent delegation.

.github/agents/db-expert.agent.md
--- # YAML frontmatter — declarative config name: DB Expert description: Database design, queries & migrations tools: [codebase, editFiles, runCommand, fetch] model: claude-sonnet-4-5 (copilot) handoffs:   - label: Generate Migration     agent: migration-expert     prompt: Generate a Prisma migration for the schema above --- ## DB Expert Agent You are a database expert focused on PostgreSQL and Prisma ORM. When analysing schemas, use #tool:codebase first. Always add indexes for foreign keys and high-cardinality columns. Migrations must be reversible — always include down steps. ## Extension API (programmatic) const agent = vscode.chat   .createChatParticipant('myorg.db-agent',     async (req, ctx, stream, token) => {       const resp = await req.model         .sendRequest(messages, {}, token);       for await (const chunk of resp.text)         stream.markdown(chunk);   });
