// Shell commands at agent lifecycle points

Config: .github/hooks/*.json

Declare hooks in JSON: type: "command" with a shell script path. VS Code auto-discovers files in .github/hooks/, .claude/settings.json, or ~/.claude/settings.json.

8 Lifecycle Events

SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, PreCompact, SubagentStart, SubagentStop, Stop — each fires at the corresponding agent session stage.

Block via Exit Code or JSON Output

Exit 2 to block with error. Return JSON {"{"}"permissionDecision": "deny"{"}"} in PreToolUse to reject dangerous commands. Use additionalContext to inject guidance.

Configure with /hooks command

Type /hooks in Chat to open an interactive UI for selecting event types, adding entries, and opening config files in the editor.

{   "hooks": {   // Block rm -rf before it runs     "PreToolUse": [{       "type": "command",       "command": "./scripts/validate-tool.sh",       "timeout": 10     }],   // Auto-format after every file edit     "PostToolUse": [{       "type": "command",       "command": "npx prettier --write \"$TOOL_INPUT_FILE_PATH\""     }],   // Inject project context at session start     "SessionStart": [{       "type": "command",       "command": "./scripts/inject-context.sh"     }]   } }
