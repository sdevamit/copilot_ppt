I'm seeing high latency in the processOrder function. How can I optimize it?

Looking at the code, you're making sequential database calls inside a loop. We can optimize this using Promise.all for concurrency.

// Before: Sequential (Slow)
for (const item of items) {
await db.updateInventory(item);
}
// After: Concurrent (Fast)
await Promise.all(
items.map(item => db.updateInventory(item))
);

Good catch. But what if one update fails? I don't want partial updates.

In that case, you should wrap the operations in a database transaction to ensure atomicity.
