Context: auth/service.ts

How should I implement refresh tokens here? Compare rotating vs sliding sessions. Which is safer?

GitHub Copilot

Based on your auth service, here is a comparison:

1. Rotating Tokens (Recommended)
Each use invalidates the old token. Higher security because stolen tokens expire quickly.

2. Sliding Sessions
Extends expiration on activity. Better UX, but a stolen token remains valid longer.

Proposed Implementation:

interface TokenConfig {
  mode: 'rotating';
  // 15m window to handle race conditions
  gracePeriod: '15m';
}
function validateToken(token) {
  if (token.isReused()) {
    // Security: Revoke entire family
    return this.revokeFamily(token.sub);
  }
}
