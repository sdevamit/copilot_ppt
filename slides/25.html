// Give Copilot Agent custom capabilities

Step 1: contributes.languageModelTools in package.json

Declare `name`, `modelDescription`, `inputSchema`, and optional `when` clause. This static config powers agent discovery — required before registerTool().

Step 2: vscode.lm.registerTool(name, impl)

Implement the LanguageModelTool<T> interface: prepareInvocation() for confirmation messages and invoke() to return a LanguageModelToolResult.

MCP Tools — No Extension Code Needed

Register an MCP server in .vscode/mcp.json. All MCP server tools are auto-exposed to Copilot Agent — no TypeScript extension required.

package.json + tools/runTests.ts
// Step 1 — package.json static declaration (required) "contributes": {   "languageModelTools": [{     "name": "myorg_runTests",     "displayName": "Run Tests",     "canBeReferencedInPrompt": true,     "modelDescription": "Run the Vitest suite and return pass/fail results",     "inputSchema": {       "type": "object",       "properties": {         "pattern": { "type": "string", "description": "Test file glob" }       }     }   }] } // Step 2 — TypeScript implementation context.subscriptions.push(   vscode.lm.registerTool('myorg_runTests', {     async invoke({ input }, token) {       const result = await runVitest(input.pattern);       return new vscode.LanguageModelToolResult(         [new vscode.LanguageModelTextPart(JSON.stringify(result))]       );     }   }) );
Agent invokes automatically when fixing tests:
✓ Tool call: myorg_runTests { pattern: "**/*.test.ts" } ✓ 42 passed, 1 failed → Agent reads output & self-corrects
