Critical Awareness

Security, Privacy & Responsible Use

You are the last line of defense. Review everything.

Copilot suggestions are probabilistic, not authoritative. Always verify generated code before shipping to production.

DO: Review for Vulnerabilities

Scan for SQL injection, XSS, insecure deserialization, and weak cryptography in generated code. Use SAST tools as a second check.

DO: Use Content Exclusions

Configure .copilotignore to exclude sensitive files (e.g., secrets, internal schemas) from Copilot context.

DON'T: Paste Secrets

Never include API keys, passwords, tokens, or PII directly in prompts or in code sent to Copilot. Use environment variables instead.

DON'T: Ignore License Flags

Copilot can detect and flag suggestions matching public code. Review flagged suggestions for license compatibility (MIT, GPL, etc.) before using.

Data Privacy Facts

Code is NOT used to train models

With Copilot for Business/Enterprise, your code snippets are not retained or used for model training by GitHub.

Prompts sent to GitHub's servers

Your code context (snippets + surrounding code) is sent to GitHub's AI infrastructure over TLS-encrypted connections.

Enterprise: Org-level policies

Admins can restrict which repos and file patterns Copilot can read, providing compliance and IP protection controls.

Duplication detection filter

Copilot can optionally filter out suggestions that match verbatim blocks of public code, protecting against unintended license obligations.

No output guarantee

Suggestions may contain security anti-patterns. Run SAST (CodeQL, Snyk) on all AI-generated code as part of your CI pipeline.

See GitHub's Privacy Statement and Copilot Trust Center for full terms.
